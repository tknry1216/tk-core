# Go言語トレンドサマリー (2026年4月)

## 1. 最新バージョン: Go 1.26 (2026年2月10日リリース)

### 言語仕様の変更

- **`new` 関数の拡張**: `new` の引数に式を指定して初期値を設定可能に
- **ジェネリクスの自己参照**: ジェネリック型が自身の型パラメータリスト内で自分自身を参照可能に。複雑なデータ構造やインターフェースの実装が簡素化

### パフォーマンス改善

- **Green Tea GC (ガベージコレクタ)**: 実験的だったGreen Tea GCがデフォルトで有効化。オブジェクト中心の並行マーク&スイープからメモリブロック中心アーキテクチャへ移行し、小オブジェクトのマーキング・スキャン性能が大幅向上。実世界のプログラムでGCオーバーヘッドが **10%〜40%削減**
- **cgoオーバーヘッド約30%削減**: cgo呼び出しのベースラインコストが大幅に低減
- **スライスのスタック割り当て最適化**: コンパイラがより多くのケースでスライスのバッキングストアをスタックに割り当て可能に

### 新機能・実験的機能

- **実験的SIMDサポート (`simd/archsimd`)**: `GOEXPERIMENT=simd`で有効化。amd64アーキテクチャで128/256/512ビットベクトル型 (例: `Int8x16`, `Float64x8`) をサポート
- **AVX-512活用**: ランタイムがGC中の小オブジェクトスキャンにAVX-512命令を活用
- **`crypto/hpke`**: 新しい暗号パッケージ
- **実験的goroutineリークプロファイル**: goroutineリークの検出を支援
- **`go fix` の刷新**: コードベースを最新のイディオムやコアライブラリAPIへ更新するための信頼性の高いツールに

### Go 1.24 / 1.25 からの主要機能 (振り返り)

| バージョン | 主要機能 |
|-----------|---------|
| Go 1.24 (2025/02) | ジェネリック型エイリアス、弱参照 (`weak`パッケージ)、Swiss Tableベースmap、FIPS 140-3準拠暗号 |
| Go 1.25 (2025/08) | ジェネリクスのコア型簡素化、PGO改善、ランタイム性能向上 |

### 標準ライブラリ (`net/http`)

- Go 1.22で強化されたルーティングパターン (`GET /users/{id}`) により、標準ライブラリのみでのAPI開発がさらに実用的に
- Go開発者の間で最も一般的なルーティング選択肢

### データベース・ORM

- **sqlc**: SQLファーストの型安全コード生成。Go哲学に合致し、トレンド上昇中
- **GORM**: 依然として最も使われるORM
- **Ent** (Meta): コード生成ベースORM、グラフ機能
- **pgx**: PostgreSQLドライバのデファクト
- **Bun**: 軽量ORM、2026年注目ライブラリの一つ
- **Atlas**: データベーススキーマ管理ツール

### 可観測性・テスト

- **OpenTelemetry Go SDK**: トレース・メトリクス・ログの標準
- **slog** (標準ライブラリ): 構造化ロギングのデファクト (Go 1.21〜)
- **golangci-lint**: 100以上のリンターを含むオールインワンツール。CI/CDの標準
- **Testcontainers-go**: コンテナを使った統合テストの標準ツールに成長
- **testify**: 27%の開発者が利用。テストの可読性向上

### CLI・TUI

- **Cobra + Viper**: CLIフレームワークの定番
- **Bubble Tea** (Charm): TUIフレームワーク。Charmエコシステム (Lip Gloss, Huh, VHS等) が拡大中

## 3. クラウドネイティブ・インフラストラクチャ

Go はクラウドネイティブ領域の **主要言語** であり続けている。

### Kubernetes エコシステム

- **client-go / controller-runtime / kubebuilder**: Operator開発の標準スタック
- **Karpenter**: Cluster Autoscalerを置き換えるノードオートスケーラー
- **Gateway API**: GA卒業、コントローラはGoで実装

### コンテナ・ネットワーキング

- **containerd**: 業界標準コンテナランタイム
- **Cilium**: eBPFベースネットワーキング (Go実装)
- **ko**: Go向けコンテナビルドツール

### ワークフロー・CI/CD

- **Temporal**: ワークフローオーケストレーション (Go SDKが人気)
- **Dagger**: CI/CDをコードとして記述。Go製・Go SDK提供

### サーバーレス

- Lambda / Cloud Functions でのGoランタイムは高速コールドスタートを実現し、トップティアの選択肢

## 4. Go と AI/ML

### AI インフラストラクチャとしてのGo

Goは **AIエージェント基盤** および **推論サービング層** として急速にポジションを確立。

- **MCP (Model Context Protocol) Go SDK v1.0.0**: 公式SDKがリリース (`github.com/modelcontextprotocol/go-sdk/mcp`)。MCPはLinux Foundationに寄贈され、ベンダー中立なガバナンスの下で運営
- **Ollama**: Go製のローカルLLM推論サーバー。100,000+ GitHub Stars。2024-2025年のブレイクスルースター
- **LocalAI**: OpenAI互換API (Go製)
- **LocalAGI**: セルフホスト型AIエージェントプラットフォーム (Go製)。Discord/Slack/GitHub統合
- **LangChainGo**: LangChainのGoポート。チェーン・エージェント・ベクトルストア
- **Weaviate**: Go製ベクトルデータベース

### GoのAI領域での役割

- AIモデルの **トレーニング** は引き続きPython領域 (PyTorch/JAX等)
- Goの強みは **推論サーバー、APIレイヤー、オーケストレーション層、MCPサーバー** 構築
- AI/MLワークロード周辺のインフラ (キュー、ルーティング、ストリーミング) に最適

## 5. コミュニティ・採用動向

### TIOBE Index

- 2025年: **#7〜#8** 付近で推移
- 2026年1月: **#16** に下落
- 下落はAIコードアシスタントがPython等のより人気の高い言語を優遇する傾向や、ランキング上位の統合が要因
- 産業的価値やエンタープライズ採用には影響なし

### 採用・求人市場

- Go開発者のポジションは **年間15%成長**
- 金融サービス (Goldman Sachs, Capital One等) での採用拡大
- プラットフォームエンジニアリング・SRE・DevOps領域で強い需要
- 給与水準は一貫して上位

### 開発者調査 (JetBrains / Stack Overflow)

- Stack Overflow: 「最も賞賛される言語」トップ5〜7に安定
- Go開発者の割合: 約14% (成長傾向)
- JetBrains: 年間2〜3%の採用率成長

## 6. 注目すべきアーキテクチャパターン

- **モジュラモノリス**: まずモノリスで開始し、後から分割。Goのパッケージシステムが支援
- **ConnectRPC**: gRPC互換のモダンRPCフレームワーク (Buf社)。REST + gRPCの二刀流
- **イベント駆動 (NATS JetStream / Watermill)**: Goでのイベントストリーミングの定番に
- **WebAssembly**: `GOOS=wasip1` (WASI Preview 1) サポート。Wazero (Pure Go WASMランタイム) が人気

## 7. まとめ

Go 1.26は Green Tea GC、SIMD実験サポート、cgo高速化など **パフォーマンス面での大きな進化** を遂げた。エコシステムでは標準ライブラリの強化により外部依存を減らす方向が継続しつつ、AIエージェント開発 (MCP SDK) やクラウドネイティブ領域での存在感がさらに拡大している。TIOBE Indexでのランキング下落はあるものの、エンタープライズ採用と求人市場は堅調であり、**インフラ・バックエンド・AI基盤** の言語としてのGoのポジションは2026年もゆるぎない。
