# Go言語トレンドサマリー (2025-2026)

調査日: 2026-03-23

---

## 1. 最新バージョンと主要機能

### Go 1.24 (2025年2月)

- **ジェネリック型エイリアス**: 型エイリアスのパラメータ化を完全サポート
- **`go.mod` のツールディレクティブ**: `tools.go` ワークアラウンドが不要に。`go get -tool` で追加可能
- **Swiss Tables マップ実装**: Go 1.23比で最大60%高速化
- **弱ポインタ（Weak Pointers）とクリーンアップファイナライザの改善**
- **FIPS 140-3ネイティブサポート**: 暗号ライブラリに組み込み
- **ランタイム改善**: CPU オーバーヘッド平均2-3%削減、小オブジェクトのメモリ割り当て効率化、新しいランタイム内部mutex
- レジスタベースABI以来最も重要なパフォーマンスリリースと評価

### Go 1.25 (2025年8月)

- **JSON v2（実験的）**: `encoding/json/v2` および `encoding/json/jsontext` パッケージ（`GOEXPERIMENT=jsonv2`で有効化）
- **testing/synctest**: 実験的機能からGA（一般提供）に昇格
- **Green Tea GC（実験的）**: 新しいガベージコレクタを実験的に導入
- **ビルド改善**: `go build -asan` でプログラム終了時のリーク検出がデフォルトに

### Go 1.26 (2026年2月) — 最新

- **`new(expr)` 構文**: `new` 関数が初期値を指定する式を受け付けるように
- **自己参照ジェネリック型**: ジェネリック型が自身の型パラメータリスト内で自己参照可能に
- **Green Tea GCがデフォルト化**: 実世界のプログラムでGCオーバーヘッド10-40%削減。新しいamd64 CPU（Ice Lake+, Zen 4+）ではベクトル命令により更に約10%改善
- **cgoオーバーヘッド約30%削減**
- **スライスのスタック割り当て改善**: コンパイラがより多くの状況でスタック上にバッキングストアを確保
- **実験的SIMD**: `simd/archsimd` パッケージ
- **実験的 `runtime/secret`**: 暗号処理の一時データの安全な消去
- **実験的ゴルーチンリークプロファイル**: `runtime/pprof` に追加
- **`go fix` の刷新**: 新しい言語機能を活用したコードモダナイゼーション

## 2. 人気フレームワーク・ライブラリ

### Webフレームワーク

| フレームワーク | シェア | 特徴 |
|--------------|--------|------|
| **Gin** | ~48% | 最も成長が速い。ミニマリストで高パフォーマンスなREST API |
| **Echo** | ~16% | 機能豊富、マイクロサービスに人気 |
| **Fiber** | ~11% | fasthttp基盤。Node.js開発者に親和性が高い |
| **Chi** | 成長中 | 軽量ルーター、標準ライブラリ互換 |
| **Hertz** | 新興 | ByteDance製。高スループットシステム向け |
| **Encore.go** | 新興 | 型安全な分散システム、組み込みオブザーバビリティ |
| **Kratos** | 新興 | Bilibili発のエンタープライズマイクロサービス |

### 主要ライブラリ

- **ロギング**: `log/slog`（標準ライブラリ、1.21以降）がデフォルトに。高パフォーマンス向けには `zap`、`zerolog`
- **データベース**: `sqlx` が `database/sql` 上で引き続き人気
- **リンター**: `golangci-lint` がデファクトスタンダード（100以上のリンター内蔵、並列実行、キャッシュ対応）
- **Nil安全**: `nilaway` によるnilパニック検出

### 注目すべき変化

- `gorilla/mux` は2023年にアーカイブ化。`chi` や標準ライブラリ（Go 1.22で `http.ServeMux` にパターンルーティング追加）への移行が進行中
- `net/http` 標準ライブラリがシンプルなサービスには十分な機能を提供

## 3. エコシステム・ツーリングの変化

### ツーリング

- **Go Telemetry** (1.23で導入): `go`、`gopls`、`govulncheck` をカバー
- **Profile-Guided Optimization (PGO)**: Uber + Google協力。go-jsonのiTLBミス30%削減、インライン化最適化で約4%パフォーマンス向上
- **gopls MCPサーバー統合（実験的）**: AIツール統合のためのIDEエージェントプロトコル
- **Go MCP SDK**: Anthropicとの協力でリリース。MCPは2025年12月にLinux Foundation (AAIF) に寄贈

### TypeScriptコンパイラのGo書き換え

- TypeScriptチームがコンパイラのGo書き換えを選択
- 2025年半ばのマイルストーンを過ぎ、早期プレビューが利用可能。2026年に広範な本番利用が見込まれる

### セキュリティ

- 2025年10月に1回のリリースで10件のセキュリティ修正
- Trail of Bits監査がGoの暗号化を検証
- サプライチェーンリスク: BoltDBのタイポスクワット、偽MongoDBドライバがモジュールエコシステムで確認

|------|------|
| TIOBE Index順位 | 7位（2025年4月）— Go史上最高位 |
| JetBrains Language Promise Index | 4位（TypeScript, Rust, Pythonに次ぐ） |

| Cloudflare Radar API呼び出し | 全自動APIリクエストの12%（Node.jsを超え1位） |

### クラウドネイティブ

- Kubernetes、Docker、Prometheus、Terraform、Nomad — すべてGoで記述
- Uber、Netflix、Google、Dropboxのマイクロサービスアーキテクチャの中核

- Go開発者の約22%がAI/LLM機能に関与（2025-2026年）
- モデルのトレーニングではなく、オーケストレーションとサービングに活用（Pythonを補完）
- **LangChain Go**: 20以上のプロバイダーサポート
- AI関連ライブラリ: chromem-go（ベクトルDB）、goai（20+プロバイダー）、hotplex（AIエージェントランタイム）
- GoのAIエージェントフレームワークエコシステムは「興味深い実験から本格的な本番オプション」に成熟

### WebAssembly

- ブラウザ・エッジワークロード向けのWASMコンパイルが着実に普及
- Kubernetesオペレーター/サイドカー、クロスプラットフォームツール、エッジコンピューティング

### エッジコンピューティング / ARM64

- Go開発者の約50%がARM64をターゲットに
