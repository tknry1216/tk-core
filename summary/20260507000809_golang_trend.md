# Go言語トレンドサマリー (2026年5月)

## 1. Go 1.26 リリース (2026年2月)

Go 1.25に続き、2026年2月にGo 1.26がリリースされた。主な変更点は以下の通り。

### 言語仕様の変更

- **`new`関数の拡張**: `new`のオペランドに式を指定でき、変数の初期値を直接設定可能に
- **ジェネリクスの自己参照**: ジェネリック型が自身の型パラメータリスト内で自分自身を参照可能に。複雑なデータ構造やインターフェースの実装が簡素化

### パフォーマンス改善

- **Green Tea GC**: 実験的だったGreen Teaガベージコレクタがデフォルトで有効化
- **cgoオーバーヘッド削減**: cgoのベースラインオーバーヘッドが約30%削減
- **スタック割り当ての最適化**: コンパイラがスライスのバッキングストアをスタックに割り当て可能なケースが拡大

### 開発者ツール

- **`go fix`の刷新**: Goの解析フレームワークを使って完全に書き直され、数十の「modernizer」アナライザが含まれるように。古いコードを安全に新しい言語機能に移行するための修正を提案

### 標準ライブラリ

- `crypto/hpke`パッケージの追加
- `crypto/mlkem/mlkemtest`パッケージの追加
- `testing/cryptotest`パッケージの追加

## 2. Go 1.24 / 1.25 の重要な機能 (振り返り)

### Go 1.24 (2025年2月)

- **ジェネリック型エイリアス**: 型エイリアスが定義型のようにパラメータ化可能に
- **ツールディレクティブ**: `go.mod`に`tool`ディレクティブが追加され、従来の`tools.go`ハックが不要に
- **弱参照 (Weak Pointers)**: メモリ管理の高度なユースケースに対応
- **高速Map実装**: Map実装の刷新によるパフォーマンス向上
- **WebAssembly対応強化**: `go:wasmexport`ディレクティブの追加
- **ポスト量子鍵交換**: X25519MLKEM768がデフォルトで有効

### Go 1.25 (2025年8月)

- セキュリティ修正とバグ修正を中心としたリリース
- `archive/tar`, `crypto/tls`, `crypto/x509`, `html/template`, `os`パッケージのセキュリティ修正

## 4. AI・クラウドネイティブ領域での台頭

### AI基盤としてのGo

- **モデルサービング**: kServe、Firebase GenKitなどがGoでモデルをプロダクション運用
- **AIエージェントフレームワーク**: ADK、Genkit、LangChain Go、Eino、Anyiなど多数登場
- **プロトコル対応**: Agent2Agent (A2A)、Model Context Protocol (MCP) のGo実装が普及
- **パフォーマンス優位**: Pythonベースと比較してレイテンシ30%改善、スループット25%向上

### クラウドネイティブの基盤

- Kubernetes、DockerなどのCNCFエコシステムの中核言語としての地位が安定
- マイクロサービス、分散システム、リアルタイムアプリケーションでの採用拡大

### AI支援開発

- Go開発者の70%以上がAIアシスタント・エージェント・AIコードエディタを日常的に利用
