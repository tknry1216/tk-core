# Go言語トレンドサマリー (2026年5月)

## 1. 最新リリース状況

### Go 1.26 (2026年2月リリース)

Go 1.26は2026年2月10日にリリースされた最新の安定版。主な変更点は以下の通り。

- **言語仕様の改善**: `new`関数のオペランドとして式を指定可能に。ジェネリック型が自身の型パラメータリスト内で自己参照可能となり、複雑なデータ構造やインターフェースの実装が簡素化
- **Green Tea GC（デフォルト化）**: Go 1.25で実験的に導入されたGreen Tea GCがデフォルトで有効化。従来のオブジェクト中心の並行マーク＆スイープから、メモリブロック中心のアーキテクチャに移行。小さいオブジェクト（< 512バイト）を8KiBスパン単位で処理し、ランダムなポインタ追跡を逐次スキャンに変換。実運用で**GCオーバーヘッドを10〜40%削減**
- **SIMD対応**: Intel Ice Lake / AMD Zen 4以降ではベクトル命令による並列スキャンが可能で、さらに約10%の性能向上
- **cgoオーバーヘッド約30%削減**: cgo呼び出しのベースラインオーバーヘッドを大幅に削減
- **スタック割り当ての最適化**: スライスのバッキングストアをスタック上に割り当てるケースが拡大
- **実験的パッケージ**: `simd/archsimd`（SIMD操作）、`runtime/secret`（シークレット管理）

### Go 1.25 (2025年8月リリース)

- 実験的なGreen Tea GC初搭載
- 実験的な`encoding/json/v2`パッケージ

### Go 1.27 (2026年8月リリース予定)

- レスポンスファイル（@file）パースのサポート追加

## 2. 言語機能の進化

### Range-over-Function (Go 1.23〜)

Go 1.23で導入されたrange-over-functionイテレータは、Goの反復処理を大きく変えた機能。

- `for/range`文が関数型に対応し、カスタムイテレーションロジックの定義が可能に
- 標準ライブラリに`iter`パッケージ（`Seq`/`Seq2`型）を追加
- ジェネリクスとの統合により、型安全なイテレーションが容易に
- メモリ効率の向上: スライスを返す従来の関数を、値を逐次生成するイテレータベースに置換可能

### ジェネリクスの深化

- Go 1.26でジェネリック型の自己参照が可能に
- 標準ライブラリ全体でジェネリクスの活用が進行中

## 4. AI/ML エコシステム

Go言語のAI/ML分野への進出が加速しており、2026年の注目トピックとなっている。

### LLMアプリケーションフレームワーク

- **LangChainGo**: PythonのLangChainをGoに移植。チェーンベースのAIアプリ開発
- **Eino (CloudWeGo)**: Go規約に従ったLLMアプリケーション開発フレームワーク
- **Google GenKit**: マルチモデル対応の統一AI API。マルチモーダル、ツール呼び出し、エージェントワークフロー対応
- **Google ADK / Jetify AI SDK / Anyi**: AIエージェント構築用フレームワーク

### ローカルLLM実行

- **Ollama**: Goで書かれたローカルLLMサーバー。Llama 3、Mistral、Gemmaなどをローカル実行
- **LocalAI**: OpenAI API互換のローカル推論エンジン。GPU不要

### MCP (Model Context Protocol)

- **MCP Go-SDK（公式）** / **MCP-Go（コミュニティ）**: LLM統合の標準プロトコルをGoで実装

### マルチプロバイダLLMインターフェース

- **any-llm-go (Mozilla AI)**: OpenAI、Claude、Mistral等を統一インターフェースで利用可能

## 5. WebAssembly (WASM) サポート

### 現状

- Go 1.24で`go:wasmexport`ディレクティブ追加。Go関数をWasmホストからアクセス可能に
- WASIリアクタービルドフラグにより、長期実行アプリケーションでエクスポート関数を維持可能

### WASIp3の展望

- WASIp3（WASI 0.3）の提案が進行中。ゴルーチンのイディオマティックな利用をサポートする初のWASIマイルストーン
- あるゴルーチンがI/Oやsleepでブロックしても、他のゴルーチンをブロックしない
- Wasmtime ランタイムでWASI 0.3の実験的サポートが2026年2月から開始、2026年末〜2027年初の1.0リリースが目標

## 6. セキュリティ動向

### サプライチェーンセキュリティ

- **BoltDBタイポスクワット攻撃**: 偽のBoltDB/MongoDBドライバーを通じたサプライチェーン攻撃が発覚し、Googleがモジュールプロキシおよび脆弱性DBで対応
- **govulncheck**: CI/CDでの有効化が推奨。既知の脆弱性を検出
- **Capslock**: ケイパビリティ分析ツール。パッケージレベルの権限追跡
- **GoLeash**: サプライチェーン緩和のための研究。パッケージ単位での細粒度ポリシー適用

### 暗号ライブラリ

- Trail of Bits による暗号監査で安全性を検証済み
- 2025年にはセキュリティ修正を含むポイントリリースが頻繁に実施（10月に10件のセキュリティ修正を含むリリースなど）

### 主な脆弱性対応

- CVE-2025-58181 / CVE-2025-47914: `golang.org/x/crypto/ssh`の無制限メモリ消費・パニック → v0.45.0で修正
- CVE-2025-61731: `cmd/go`でのpkg-config経由の任意ファイル書き込み
- CVE-2026-25679: IPv6ホストリテラルの不正パース

## まとめ

2026年のGo言語は、**Green Tea GCによる大幅なパフォーマンス向上**、**AI/LLMエコシステムの急速な発展**、**WASIp3によるWebAssembly統合の深化**が三大トレンドである。言語自体は安定した進化を続けつつ、エコシステムがクラウドネイティブ・AI領域へ急速に拡大している。特にOllama、LangChainGo、Einoなど、GoによるAIアプリケーション開発基盤の充実が顕著であり、Goの「高性能バックエンド言語」としてのポジションがAI時代にも強く維持されている。
