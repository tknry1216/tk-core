# Go言語トレンドサマリ (2026年4月)

## 1. 最新リリース

### Go 1.24 (2025年2月)
- **ジェネリック型エイリアス**: パラメータ化された型エイリアスを完全サポート
- **ツール依存管理**: `go.mod` に `tool` ディレクティブ導入（従来の `tools.go` ブランクインポートを廃止）
- **WebAssembly**: `go:wasmexport` ディレクティブでGoの関数をWasmホストにエクスポート可能に
- **Swiss Tables マップ実装**: マイクロベンチマークで最大60%のマップ操作高速化
- **FIPS 140-3暗号モジュール**: ポスト量子 ML-KEM 鍵交換（X25519MLKEM768）を含むネイティブ対応
- CPU オーバーヘッド平均2-3%削減

### Go 1.25 (2025年8月)
- **Green Tea GC (実験的)**: `GOEXPERIMENT=greenteagc` でオプトイン
- **encoding/json/v2 (実験的)**: `GOEXPERIMENT=jsonv2` でオプトイン。高速アンマーシャリング、不正UTF-8・重複キーのデフォルト拒否
- **コア型概念の廃止**: ジェネリクス仕様を型セット＋明示的記述に簡素化
- **DWARF 5**: デフォルト化によりバイナリのデバッグ情報サイズ削減
- **`sync.WaitGroup.Go()`**: goroutine生成とカウントを簡潔に記述可能に

### Go 1.26 (2026年2月)
- **`new(expr)` 構文**: `new` ビルトインが式を引数に取れるように（初期値指定付きアロケーション）
- **自己参照ジェネリック制約**: 型パラメータリスト内での自己参照が可能に
- **Green Tea GCがデフォルト有効化**: GCヘビーなワークロードで10-40%のGCオーバーヘッド削減。amd64でAVX-512ベクトル命令を活用
- **cgoオーバーヘッド約30%削減**
- **スタック上スライス割り当て**: コンパイラがより多くのケースでスライスバッキングストアをスタックに割り当て
- **`go fix` 刷新**: 汎用コードモダナイザーとして全面書き直し
- **新標準ライブラリ**: `crypto/hpke`, `crypto/mlkem/mlkemtest`, `testing/cryptotest`
- **実験的パッケージ**:
  - `simd/archsimd`: amd64向け128/256/512ビットSIMD操作
  - `runtime/secret`: 機密データの安全な消去
  - `runtime/pprof` にgoroutineリークプロファイル

## 8. セキュリティ

### FIPS 140-3 準拠
- Go 1.24で標準ライブラリにネイティブFIPS 140-3暗号モジュールを搭載
- ポスト量子 ML-KEM (FIPS 203) 鍵交換: `crypto/tls` でFIPS 140-3準拠のポスト量子セキュア接続が可能
- **重要期限**: FIPS 140-2が**2026年9月21日にサンセット**。以降はFIPS 140-3モジュールのみ有効

### 脆弱性管理
- **govulncheck**: ツールチェーンに統合。コールグラフ解析により実際に到達する脆弱性のみを検出（低ノイズ）
- Go脆弱性データベースがキュレーション済みレポートを継続的に更新
- **BoltDBタイポスクワット事件**: `github.com/boltdb-go/bolt` がGoモジュールプロキシキャッシュ経由で3年以上存続した事例が脆弱性DBに記録

### Go 1.26の新セキュリティ機能
- `crypto/hpke` (Hybrid Public Key Encryption)
- `crypto/mlkem/mlkemtest` (ポスト量子テスト)
- `runtime/secret` (実験的: 暗号一時データの安全な消去)

### サプライチェーンセキュリティ
- Goモジュールシステム + チェックサムDB (`sum.golang.org`) + モジュールプロキシが堅固な基盤を維持
- **Capslock**: Googleが推奨するケーパビリティ分析ツール
