## 1. 最新リリース動向

### Go 1.26（2026年2月リリース）

Go 1.26 は 2026年2月に Go 1.25（2025年8月）から 6か月後のスケジュールでリリースされた最新安定版。小さな言語仕様の改善に加え、ランタイム／GCの大幅なパフォーマンス改善、暗号・セキュリティ領域の拡充、ツールチェインの大刷新が行われた。

#### 言語仕様の変更
- **`new` 組み込み関数の拡張:** `new(T)` の引数に「型」だけでなく「式」を受け取れるようになり、`p := new(expr)` のように初期値付きでポインタを生成可能に。
- **自己参照ジェネリクス:** ジェネリック型が自分自身を型パラメータ制約内で参照できるようになり、再帰的なデータ構造・インターフェース制約の記述が大幅に簡潔化。

#### ランタイム／パフォーマンス
- **Green Tea GC が既定で有効化:** Go 1.25 で実験的導入された新GCが正式採用。小オブジェクトのマーク／スキャン局所性と CPU スケーラビリティが改善され、GCを多用する実アプリで **GCオーバーヘッドを 10〜40% 削減**。
- **cgo オーバーヘッド 約30% 削減:** cgo 呼び出しのベースラインコストが大幅に改善、C 連携コードの性能が向上。
- **スライス裏付けのスタック割当:** コンパイラがより多くのケースでスライスの裏付け領域をスタック上に割当可能になり、ヒープ圧を低減。
- **SIMD によるGC加速:** amd64（AVX-512）上で小オブジェクトのスキャンをベクトル命令で高速化し、スキャンフェーズを +10% 短縮。

#### 新規／強化パッケージ
- **`crypto/hpke`:** RFC 9180 準拠の Hybrid Public Key Encryption を標準ライブラリで提供。ポスト量子ハイブリッド KEM もサポート。
- **`crypto/mlkem/mlkemtest`, `testing/cryptotest`:** 暗号実装テスト用ユーティリティを追加。
- **`simd/archsimd`（実験・`GOEXPERIMENT=simd`）:** amd64 向けの 128/256/512 bit ベクトル型（`Int8x16`, `Float64x8` 等）と演算を提供する実験的 SIMD API。

#### ツールチェイン
- **`go fix` の全面刷新:** `go/analysis` フレームワーク上に再実装され、20 超の「modernizer」アナライザを同梱。新しい言語機能・標準ライブラリへ安全に自動移行可能に。
- **Go 1.24／1.25 からの継続機能:** 汎用型エイリアス（1.24）、`tool` ディレクティブによる CLI ツール依存管理（1.24）、`GOEXPERIMENT=jsonv2` 新JSON実装（1.25）、`sync.WaitGroup.Go`（1.25）、cgroup 対応 `GOMAXPROCS`（1.25）。

## 2. エコシステム・周辺ツール

### 標準リンタ／品質ツール
- **golangci-lint** が CI/CD・ローカル双方で事実上の標準に。100 以上のリンタを並列実行・キャッシュ付きで統合し、2026 年時点ではほぼすべての主要 OSS プロジェクトで採用。

### 主要フレームワーク
- **Web:** Gin / Echo / Fiber が引き続きトップ3。軽量 API・マイクロサービス・AI ゲートウェイ用途で採用拡大。
- **RPC/通信:** gRPC v1.54（2026年）は AI ワークロード向けの拡張（ストリーミング最適化・トレース連携）を導入、Go 実装がリファレンス的存在。
- **クラウドネイティブ:** Kubernetes / Terraform / etcd / Prometheus など基盤 OSS の主要実装言語としての地位は磐石。

### IDE／エディタ
- **GoLand 2026.1（JetBrains, 2026/3 リリース）:** Go 1.26 対応、AI アシスタント統合強化、`go fix` modernizers との UI 連携。
- VS Code + Go 拡張 + `gopls` がオープンソース陣営の標準。

## 3. AI / エージェント領域での採用

2026 年の最大のトピックは **Go の「AIエージェント基盤言語」としての台頭**。

- **MCP (Model Context Protocol):** Anthropic 提唱の MCP が 2026 年 3 月に累計インストール 9,700 万件を突破。Go は MCP サーバ実装の主要言語の一つ。
- **A2A (Agent2Agent) プロトコル:** Google 発の A2A は発表から 1 年で 150 超の組織が参加（Microsoft / AWS / Salesforce / SAP / ServiceNow 等が本番採用）、v1.0 に到達。
- **Google ADK for Go:** Google が OSS 公開した Agent Development Kit の Go 版は A2A を第一級サポートし、マルチエージェントの委譲／協調ワークフローを「通常の Go コード」として記述可能。
- **なぜ Go か:** 静的バイナリ・起動速度・軽量な並行処理（goroutine）・低メモリフットプリントが、エージェントオーケストレータ／サイドカー／MCP サーバ用途に適合。

## 4. 採用・コミュニティ動向

- **JetBrains Data Playground 調査:** Go を主言語とするプロ開発者は **約 220 万人**。5年前の約 2 倍に増加。
- **GitHub Octoverse 2024:** Go は Python / TypeScript に次ぐ **成長率 3位の言語**。2025–2026 もこの勢いを維持。
- **ユースケース分布:** クラウドネイティブ／マイクロサービスが引き続き中心。加えて CLI ツール、DevOps 基盤、AI エージェント基盤、ブロックチェーンノード実装でのシェアが拡大。
- **日本コミュニティ:** Go Conference, Go Junction, layerx.go など定期イベントが全国で活発に開催。
