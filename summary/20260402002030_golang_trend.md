# Go言語トレンドサマリー (2026年4月)

## 1. 最新リリース状況

### Go 1.26 (2026年2月リリース)

- **`new()` 関数の拡張**: 型だけでなく式を引数に取れるようになり、初期値付きの変数作成が1文で可能に
- **自己参照ジェネリクス型**: ジェネリクス型が自身の型パラメータリストで自分自身を参照可能に。複雑なデータ構造やインターフェースの実装が容易に
- **Green Tea GC の正式採用**: Go 1.25で実験的に導入されたGreen Teaガベージコレクタがデフォルトで有効化。実アプリケーションでGCオーバーヘッドが大幅削減
- **スタック上のスライス割り当て改善**: ヒープへエスケープするスライスも含め、より多くのケースでスタック割り当てが可能に
- **`ReadAll` の高速化**: 中間メモリ割り当て削減により約2倍高速化、メモリ使用量も約半分に
- **JPEG エンコーダ/デコーダ刷新**: より高速で高精度な実装に置き換え
- **WebAssembly メモリ改善**: 16MiB以下のヒープでメモリ使用量が大幅削減
- **`go fix` コマンドの刷新**: モダナイザー群を搭載し、コードベースを最新のイディオムやAPIへ自動更新可能に
- **cgo オーバーヘッド約30%削減**
- **`crypto/hpke` パッケージ**: RFC 9180準拠のHybrid Public Key Encryptionをサポート（ポスト量子ハイブリッドKEM対応）
- **`simd/archsimd` パッケージ（実験的）**: amd64向けの128/256/512ビットSIMD操作を提供

### Go 1.25 (2025年8月リリース)

- **Green Tea GC（実験的）**: 10〜40%のGCオーバーヘッド削減
- **コンテナ対応 GOMAXPROCS**: CPU制限下でのオーバースケジューリングを軽減
- **`encoding/json/v2`（実験的）**: 新しいJSON実装
- **`testing/synctest`**: 同期テストパッケージがGA化
- **`WaitGroup.Go`メソッド**: ゴルーチン生成・カウントパターンを簡略化

### Go 1.24 (2025年2月リリース)

- **ジェネリクス型エイリアス**: 型エイリアスのパラメータ化を完全サポート
- **ツールディレクティブ**: `go.mod` で実行可能な依存関係を直接追跡可能に
- **Swiss Tablesマップ**: マイクロベンチマークでGo 1.23比最大60%高速化
- **ポスト量子鍵交換**: X25519MLKEM768がデフォルト有効
- **ECHサポート**: TLSサーバーでEncrypted Client Helloをサポート

## 2. エコシステム・フレームワーク動向

### Webフレームワーク（2026年の主要選択肢）

| フレームワーク | 特徴 | GitHub Stars |
|---|---|---|
| **Gin** | 最も人気。直感的なAPI、豊富なミドルウェア | 75,000+ |
| **Fiber** | Express.jsライク。fasthttp採用で高速 | - |
| **Echo** | クリーンな構造、安定性、充実したドキュメント | - |

### 注目ライブラリ

- **OpenTelemetry**: 可観測性の標準
- **ConnectRPC**: gRPC互換のモダンRPCフレームワーク
- **sqlc**: SQLからGoコードを型安全に生成
- **Bun**: 高速ORM
- **Testcontainers**: コンテナベースのインテグレーションテスト
- **Temporal**: 耐久ワークフローエンジン
- **golangci-lint**: 100以上のリンターを並列実行するオールインワンツール

### テスティング

- 標準 `testing` パッケージが最も利用される選択肢
- **testify** (27%のGo開発者が利用): 標準テストパッケージの拡張
- **gomock** (21%): インターフェースモック

## 3. Go × AI / LLM

- PythonがAI研究の主流である一方、Goはモデルデプロイ・高性能推論環境で存在感を拡大
- **Agent2Agent (A2A)** や **Model Context Protocol (MCP)** などのAIエージェント通信プロトコルの実装言語としてGoが選好
- **LangChain Go**, **Firebase GenKit**, **kServe** などプロダクション向けモデルサービングでGoが活躍

## 4. 採用・普及状況

- Stack Overflow調査: 開発者全体の **13.5%** がGoを利用（プロ開発者では14.4%）
- JetBrains調査: 過去1年で **410万人** のプロフェッショナルがGoを使用、**180万人** が主要言語として利用
- **Cloudflare Radar**: Goが自動APIリクエストで最も利用される言語にNode.jsを超えて成長（全体の12%）
- **GitHub**: 2024年に3番目に急成長した言語（Python、TypeScriptに次ぐ）
- 主要採用企業: Google, DataDog, Dropbox, HashiCorp, Apple, Salesforce, Uber, Amazon

## 5. 今後の展望

- **Go 2.0** への進化が期待される（ジェネリクスのさらなる拡充、エラーハンドリングの簡素化など）
- クラウドネイティブ・DevOps領域での支配的ポジションが継続（Kubernetes, Docker, Terraformのエコシステム拡大と連動）
- AI/MLパイプラインのプロダクション層でのGoの役割が拡大
- `go fix` のセルフサービスパラダイムにより、ライブラリ作者が自身のAPIモダナイゼーションを定義可能に
