## Go バージョンリリース状況

### Go 1.24（2025年2月）
- **ジェネリック型エイリアス**が完全サポート（例: `type Set[T comparable] = map[T]struct{}`）
- **Weak ポインタ**（`weak` パッケージ）の導入
- **Swiss Tables** によるmap実装の刷新（マイクロベンチマークで最大60%高速化、平均CPU負荷2〜3%削減）
- **runtime.AddCleanup**: `runtime.SetFinalizer` を置き換える新しいファイナライゼーション機構
- **ポスト量子暗号**: X25519MLKEM768 鍵交換がデフォルトで有効
- **FIPS 140-3** 暗号化コンプライアンス対応
- `go tool` が `go.mod` の tool ディレクティブからツールを直接実行可能に（`tools.go` ワークアラウンド不要）
- `go build -json` フラグで構造化JSON出力に対応
- **testing/synctest**（実験的）: フェイククロックと分離goroutineによる並行コードテスト
- **testing.B.Loop**: シンプルなベンチマークイテレーション
- **os.Root**: 特定ディレクトリに制限されたファイルシステム操作

### Go 1.25（2025年8月）
- **実験的 Green Tea GC**（`GOEXPERIMENT=greenteagc`）: GCオーバーヘッド10〜40%削減
- **実験的 encoding/json/v2**: v1比で最大9倍高速なJSONエンコーディング
- `go build -asan` でプログラム終了時のリーク検出が自動化
- **コンテナ対応ランタイム**: Cgroupリミットの定期チェック（10秒間隔）とGOMAXPROCSの動的調整
- `go mod ignore` ディレクティブの追加
- `go doc -http` ローカルドキュメントサーバーの新設

### Go 1.26（2026年2月）
- **Green Tea GC がデフォルトで有効化**: 小オブジェクト集約アプリで10〜40%のGCオーバーヘッド削減
- **`new` ビルトインの拡張**: 初期値を指定可能に（例: `new(int, 42)`）
- **自己参照ジェネリック型**: ジェネリック型が自身の型パラメータリストで自身を参照可能
- 新パッケージ追加: `crypto/hpke`, `crypto/mlkem/mlkemtest`, `testing/cryptotest`
- 2026年3月に Go 1.26.1 リリース（暗号化・HTMLテンプレート・URLパッケージのセキュリティ修正）

## 主要トレンド

### 1. クラウドネイティブ領域での圧倒的存在感
- Docker、Kubernetes、Terraform、Prometheus、Istio、containerd、etcd などクラウドインフラの中核ツールがGo製
- プラットフォームエンジニアリング・社内開発者プラットフォーム（IDP）構築での採用が拡大
- バックエンド開発・クラウドインフラ・CLIツール開発で事実上の標準言語として定着
- PGO（Profile-Guided Optimization）が本番ワークロードのチューニング標準に

### 2. AI/ML インフラストラクチャへの進出
- **Ollama** の爆発的人気（GitHub 169K+スター、25億+ダウンロード）により、GoがAIインフラ言語として確立
- モデル訓練ではなく、サービング・オーケストレーション・AIアプリ構築に強み（典型バイナリ: 15〜25MB）
- LLMツール/エージェントフレームワークがGoエコシステムで台頭
- Go開発者の**70%以上**がAIアシスタント/エージェントを日常的に利用

### 3. WebAssembly（Wasm）の成熟
- WebAssembly 3.0 が W3C標準に（2025年9月）: WasmGC、例外処理、末尾呼び出し、64bitメモリ、SIMD
- Component Model（2026）: Rust・Go・Python間のWASMモジュール相互運用
- WASI 0.3（async対応）開発中、2026年末に1.0の可能性
- Go+WASMによるブラウザベースAIアプリケーションの活用事例

### 4. パフォーマンス改善
- **Swiss Tables**: Datadog社は高トラフィックサービスで「数百GBのメモリ節約」を報告
- **Green Tea GC**: 小オブジェクト集約アプリで10〜40%のGCオーバーヘッド削減、GCに10%の時間を費やすアプリで全体CPU 1〜4%削減
- **JSON v2**: v1比で最大9倍高速、slogと組み合わせて構造化ロギングが7%高速化
- PGO改善: デバーチャライゼーション強化で2〜14%の性能向上

### 5. 開発者体験の向上
- **gopls v0.17.0+**: モジュールキャッシュの永続インデックスにより Organize Imports・シンボル補完が桁違いに高速化
- 新コードアクション: 「不足メソッド宣言」「テスト生成」「定数抽出」「パラメータ移動」
- Staticcheck アナライザの大半がデフォルト有効に
- **IDE利用率**: VS Code 37%、GoLand 28%、Cursor（AI搭載）6%

## まとめ

2025〜2026年のGoは、クラウドネイティブ領域での支配的地位を維持しつつ、AI/MLインフラ（Ollama 169K+スター）やWebAssembly（W3C標準化）といった新領域への展開が加速している。言語自体もGreen Tea GCのデフォルト有効化、`new`ビルトインの拡張、自己参照ジェネリクスなど着実に進化。Swiss Tablesによるmap性能60%向上やJSON v2の9倍高速化など、パフォーマンス面でも大きな進歩を遂げている。開発者数220万人超、満足度91%と、エコシステムの健全な成長が続いている。
