## Go リリース状況

### Go 1.24（2025年2月リリース）

Go 1.24 では言語機能・パフォーマンス・セキュリティの全方位で大幅な改善が行われた。

- **ジェネリック型エイリアス**: 型エイリアスが定義型と同様に型パラメータを持てるようになった
- **ツール依存管理**: `go.mod` に `tool` ディレクティブが追加され、実行可能な依存ツールを直接追跡可能に。従来の `tools.go` ワークアラウンドが不要に
- **Swiss Tables ベースの新 map 実装**: 平均 2-3% の CPU オーバーヘッド削減
- **FIPS 140-3 準拠メカニズム**: Go Cryptographic Module により、FIPS 140-3 承認アルゴリズムを透過的に利用可能
- **ポスト量子暗号**: X25519MLKEM768 鍵交換メカニズムをサポート、デフォルトで有効
- **Encrypted Client Hello (ECH)**: TLS サーバーで ECH をサポート
- **`testing/synctest` パッケージ（実験的）**: 並行コードのテストを支援。偽の時計を使った隔離環境でゴルーチンをテスト可能
- **`testing.B.Loop`**: ベンチマークの反復をよりシンプルかつ正確に記述可能
- **`os.Root` 型**: 特定ディレクトリ内でのファイルシステム操作を安全に実行
- **`go build` / `go install` の `-json` フラグ**: 構造化 JSON 出力に対応

### Go 1.25（2025年8月リリース）

Go 1.25 では言語仕様の変更はないが、実験的な新機能と内部改善が多数含まれた。

- **実験的ガベージコレクタ（Green Tea GC）**: 小オブジェクトのマーキング・スキャンのパフォーマンスを局所性と CPU スケーラビリティで改善
- **実験的 `encoding/json/v2`**: `GOEXPERIMENT=jsonv2` で有効化可能な新 JSON 実装
- **`testing/synctest` の正式化**: Go 1.24 で実験的だった synctest パッケージが一般利用可能に
- **Core Types 概念の廃止**: Go 1.18 で導入された「Core Types」を仕様から完全に削除。現行プログラムへの影響はないが、将来の言語改善への道を開いた
- **コンパイラ強化**: `GOAMD64=v3` モード以上で FMA（Fused Multiply-Add）命令を使用し、浮動小数点演算が高速化・高精度化
- **`go build -asan` の強化**
- **`go mod ignore` ディレクティブの追加**
- **`go doc -http` サーバーの新設**
- **コンテナ対応 `GOMAXPROCS`**

### Go 1.26（2026年2月リリース）

Go 1.26 は言語改善・GC・ツーリング・セキュリティの全領域で注目すべきアップデートを含む。

- **`new` 関数の拡張**: `new` の引数に式を指定でき、初期値を直接設定可能に
- **ジェネリック型の自己参照**: ジェネリック型が自身の型パラメータリストで自分自身を参照可能に。より高度な型抽象化パターンが実現
- **Green Tea GC のデフォルト化**: Go 1.25 で実験的だった Green Tea GC がデフォルトで有効化
- **`go fix` の完全刷新**: `go vet` と同じ `analysis` フレームワーク上で再構築。非推奨 API の書き換えや古いイディオムの自動修正が可能に
- **新パッケージ**: `crypto/hpke`、`crypto/mlkem/mlkemtest`、`testing/cryptotest`
- **実験的 `simd/archsimd`**: SIMD（Single Instruction, Multiple Data）操作へのアクセスを提供
- **実験的 `runtime/secret`**: 暗号処理で使用される秘密情報の安全な破棄
- **実験的 goroutineleak プロファイル**: `runtime/pprof` でリークしたゴルーチンを検出
- **pprof Web UI**: デフォルトでフレームグラフ表示に変更
