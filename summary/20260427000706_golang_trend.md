# Go言語トレンドレポート 2026

## 1. Go言語の現状と成長

### 開発者人口
- プライマリ言語としてGoを使用するプロフェッショナル開発者は**220万人**（5年前の2倍）
- セカンダリ言語を含めると**500万人以上**
- 世界の開発者の**13.5%**がGoを選好（プロフェッショナルでは14.4%）
- GitHubで**3番目に成長が速い言語**（Python、TypeScriptに次ぐ）
- JetBrains Language Promise Indexで**4位**（TypeScript、Rust、Pythonに次ぐ）
- 全ソフトウェア開発者の**11%**が今後12ヶ月以内にGoの採用を計画

### 主な用途
- **Webバックエンド/マイクロサービス**: 190万人以上の開発者がWebサービス・ビジネスアプリを構築
- **インフラ/DevOps**: 約100万人がKubernetesなどを用いた内部プラットフォーム・CI/CDパイプライン・クラウドインフラを管理

## 2. 最新リリース: Go 1.24 / 1.25 / 1.26

### Go 1.24（2025年2月リリース）
- **ジェネリック型エイリアス**: 型エイリアスが定義型と同様にパラメータ化可能に
- **Weak Pointers & Finalizers**: 高度なメモリ管理のユースケースを容易に
- **ツールディレクティブ**: `go.mod`内で実行可能な依存関係を`tool`ディレクティブで追跡可能に（`tools.go`のワークアラウンドが不要に）
- **SHA-3サポート**およびランダムテキスト生成ユーティリティの追加

### Go 1.25（2025年8月リリース）
- **testing/synctestパッケージ**: 並行コードのテストを支援する新パッケージ。仮想化された時間（フェイククロック）を持つ隔離された「バブル」内でテスト実行が可能
- **encoding/json/v2（実験的）**: `encoding/json`パッケージの大幅な改訂版。JSONエンコーディング/デコーディングの柔軟性と性能が向上

### Go 1.26（2026年2月リリース）
- **`new()`関数の拡張**: `new`関数が任意の値を引数に取れるようになり、初期値を指定した変数作成が可能に（追加のステートメントが不要に）
- **再帰的ジェネリック型**: ジェネリック型が自身の型パラメータリスト内で自分自身を参照可能に
- **Green Tea GC**: 実験的だったGreen Teaガベージコレクタがデフォルトで有効化。パフォーマンスが大幅に向上
- **`go fix`コマンドの刷新**: モダナイザーの統合ホームとなり、コードベースを最新のイディオムやコアライブラリAPIに更新するワンボタンソリューションを提供
- **crypto/hpkeパッケージ**: RFC 9180準拠のHybrid Public Key Encryption（HPKE）を実装。ポスト量子ハイブリッドKEMをサポート
- **WebAssembly最適化**: ヒープメモリをより小さな単位で管理し、16MiB以下のヒープを持つアプリケーションのメモリ使用量を大幅に削減
- **チャネル操作・ゴルーチンスケジューリングの最適化**: 並行ワークロードの効率が大幅に向上

## 4. AI/LLM分野でのGoの台頭

### GoがAIツーリングで選ばれる理由
- 高速なコンパイル時間と効率的なランタイム
- Kubernetesとの完全な互換性（Helmチャートによるデプロイ）
- 主要クラウドプロバイダー（AWS、Azure、GCP）のGo SDKがAIモデルデプロイ・推論をサポート

### 主要なAI/LLMフレームワーク・ツール
| プロジェクト | 概要 |
|---|---|
| **Eino** (cloudwego) | LangChain/Google ADKに触発されたGoネイティブのLLMアプリケーション開発フレームワーク。ChatModel、Tool、Retrieverなど再利用可能なコンポーネントを提供 |
| **LangChainGo** | Python版LangChainのGo版。多数のLLMプロバイダー・ベクトルDBを共通APIでサポート |
| **Google Genkit for Go** | プロンプト管理・エンジニアリング、統合開発ツールによるプロダクション向け機能に注力 |
| **Ollama** | Goで書かれたオープンソースツール。コンシューマーハードウェアでLLMをローカル実行 |
| **LocalAI** | OpenAI APIのドロップインリプレースメント。GPU不要でLLM・画像生成・音声をローカル実行 |
| **Hugot** | HuggingFaceのパイプラインパターンをGoで再実装。ONNXモデルをCPU/GPU/TPUバックエンドで実行 |

### AI開発者のツール利用状況（2025年調査）
- Go開発者の**70%以上**が少なくとも1つのAIアシスタント・エージェント・コードエディタを定期的に利用
- **Cursor**（AI搭載コードエディタ）をプライマリ開発環境とする開発者が**6%**に
- AIツールへの満足度は中程度（品質面での懸念が主因）

## 5. クラウドネイティブ & DevOps

Goはモダンなクラウドネイティブ・DevOpsエコシステムの中核を担い続けている:

- **Kubernetes**: コンテナオーケストレーションのデファクトスタンダード（Go製）
- **Docker**: コンテナランタイム（Go製）
- **Terraform**: IaCツール（Go製）
- **Prometheus**: モニタリング・アラーティング（Go製）

Kubernetes・Dockerの拡大とともにGoのフットプリントも拡大し、エコシステムはGoと共にスケールしている。

## 6. 2025年Go開発者サーベイのハイライト

2025年9月に実施された調査（回答者5,379人）の主な結果:

- 開発者が求めるサポート: **ベストプラクティスの特定・適用**、**標準ライブラリの活用**、**言語・ビルトインツールのモダン化**
- Go開発者の大多数がAI搭載開発ツールを情報検索や反復的コード作成に利用
- 満足度は中程度（AI生成コードの品質に対する懸念）
