# Go言語 トレンドサマリー (2026-04-14)

本ドキュメントは、2026年4月時点におけるGo言語のトレンドをまとめたものです。言語本体の進化、フレームワーク/エコシステム、AI分野における採用動向を中心に整理しています。

## 1. 言語本体の最新動向

### Go 1.26 (2026年2月リリース)

Go 1.25 (2025年8月) から6ヶ月後にリリースされた最新バージョン。パフォーマンスとランタイムに大きな改善が入っています。

**言語仕様の変更**
- 組み込みの `new` 関数が式を引数に取れるようになり、変数の初期値を直接指定可能に。
- ジェネリック型が自身の型パラメータリスト内で自己参照可能に。複雑なデータ構造・インターフェースの実装がシンプルになりました。

**ランタイム/パフォーマンス**
- **Green Tea GC がデフォルト有効化**: Go 1.25で実験的に導入された新GCが標準動作に。小さなオブジェクトのマーク/スキャン時にロカリティとCPUスケーラビリティを改善し、GCヘビーな実アプリで **10〜40% のGCオーバーヘッド削減** を実現。
- **cgo オーバーヘッド 約30%削減**: C言語呼び出しを多用するプロジェクトで恩恵が大きい。
- **スライスのバッキングストアをスタック確保** できるケースが拡大し、ヒープアロケーションを削減。

**新規パッケージ**
- `crypto/hpke`: RFC 9180準拠の Hybrid Public Key Encryption を実装。ポスト量子ハイブリッドKEMもサポート。
- `simd/archsimd` (実験): `GOEXPERIMENT=simd` で有効化。amd64 上で 128/256/512bit ベクタ型 (`Int8x16`, `Float64x8` 等) を提供。Intel Ice Lake / AMD Zen 4+ ではGCが並列スキャンにSIMDを利用し、特定のアロケーションパターンで +10% の性能向上。

### Go 1.25 (2025年8月) のおさらい
- 実験的な新JSON実装 (`GOEXPERIMENT=jsonv2`) を提供。高速化とAPIの改善を目指す。
- Green Tea GCを実験機能として初投入。

### Go 1.24 (2025年初頭) のおさらい
- **ジェネリック型エイリアス** を完全サポート。型エイリアスも定義型と同様にパラメータ化可能に。

## 2. Webフレームワーク動向

2026年現在も、Goの主要Webフレームワークの勢力図は安定しており、以下が主要な選択肢です。

| フレームワーク | 特徴 | 立ち位置 |
| --- | --- | --- |
| **Gin** | 最も人気。GitHub 75k+ スター。豊富なミドルウェアとエコシステム。 | デファクトスタンダード |
| **Echo** | 高パフォーマンス、REST API向け、ドキュメントが充実 | Ginの対抗馬 |
| **Fiber** | fasthttpベースで低オーバーヘッド、大量リクエスト処理に強い | 高スループット用途 |
| **Chi** | `net/http` 完全互換の軽量ルータ、イディオマティック | ミニマル志向、利用率は約12%に上昇 |
| **Beego** | 総合型フルスタック | エンタープライズ色強め |

net/http 派生のシンプル系（Chi）と、独自スタックの高速系（Fiber）が両翼を占める構図は継続中。

## 3. クラウドネイティブ / インフラ領域

- Kubernetes をはじめとする主要なクラウドインフラOSSがGoで実装されている状況は変わらず、**バックエンド・インフラ・CNCF系プロジェクトの第一言語** としてのポジションは盤石。
- IoTやエッジ領域でも、静的バイナリ・低フットプリント・並行性の特性が評価され、採用が拡大。

## 4. AI / LLM エージェント領域の台頭

2026年の最大のトピックは **GoがAIエージェント開発の本命言語として急浮上** している点です。研究/学習領域はPythonが依然支配的ですが、**モデルデプロイ・高性能推論・プロダクションAIエージェント** においてGoの採用が加速しています。

### 注目のGo製AI/LLMフレームワーク (2026年版)

1. **Eino (CloudWeGo / ByteDance)**
   - ByteDance が Doubao や TikTok のレコメンド基盤で6ヶ月以上社内運用してからOSS化した、**プロダクショントラフィック実績あり** のGo-firstフレームワーク。
   - `ChatModel` / `Tool` / `Retriever` / `ChatTemplate` などの再利用可能ビルディングブロックを提供。
   - ADK (Agent Development Kit) によりツール利用・マルチエージェント協調・コンテキスト管理などのエージェントパターンを即利用可能。
2. **LangChainGo** — コミュニティ主導のLangChain Go移植。Python版と同じ思想で Chain / Loader / Retriever のライブラリを提供。
3. **Google ADK (Go版)** — GoogleのAgent Development Kit。
4. **Firebase Genkit** — Firebase経由でのAIアプリ構築向け。
5. **Jetify AI SDK** — SDKライク、シンプル志向。
6. **Anyi** / **Agent SDK Go** — 新興のエージェント特化フレームワーク。

### なぜ2026年に "Goの年" と言われるのか
- **並行処理と低レイテンシ**: エージェントが多数のツール呼び出し・外部API・ストリームを同時処理する要件にgoroutineが自然にフィットする。
- **シングルバイナリデプロイ**: コンテナ軽量化とコールドスタート短縮が、エージェント基盤のサーバレス/エッジ運用で有利。
- **型安全と保守性**: Pythonベースのエージェントコードが複雑化する中、型のある実装で長期保守を指向する企業が増加。

## 5. 長期的トレンド

- **Go 2.0への展望**: エラーハンドリングの簡素化、ジェネリクス定着後の追加改善などが引き続き議論中。既存コードを壊さない漸進的進化がGoチームの基本方針。
- **AI × Go**: TensorFlow Go bindings をはじめとしたML推論パスの強化が進行中。"学習=Python, 推論/本番=Go" の役割分担が明瞭化。
- **ポスト量子暗号**: `crypto/hpke` の追加など、標準ライブラリレベルでPQC対応が進む。

## 6. まとめ

- Go 1.26 により、**GCとcgoという実運用の痛点が一段改善** され、既存アプリの無停止バージョンアップだけでパフォーマンス向上が見込める。
- Webフレームワーク勢力図は安定、**Gin/Echo/Fiber/Chi** の4強で選定が容易。
- 最大の新潮流は **AIエージェント開発言語としてのGo**。ByteDance発のEinoなど、大規模プロダクションで実証済みのフレームワークが揃い始めた。
- 言語・標準ライブラリ・エコシステム・AI領域の4方向で、2026年のGoは "成熟と拡大" が同時に進行している。
