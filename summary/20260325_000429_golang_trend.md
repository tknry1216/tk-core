# Go言語トレンドサマリー（2025年〜2026年初頭）

調査日: 2026-03-25

---

## 1. 最新バージョンと主要機能

### Go 1.24（2025年2月リリース）

- **ジェネリック型エイリアス**: 型エイリアスにジェネリクスのパラメータを持たせることが可能に
- **Swiss Tablesベースの新map実装**: CPUオーバーヘッドを平均2〜3%削減
- **ツール依存管理**: `go.mod`に`tool`ディレクティブを追加、従来の`tools.go`ワークアラウンドが不要に
- **`os.Root`型**: 特定ディレクトリ内でのファイルシステム操作を安全に行うための新型
- **`runtime.AddCleanup`**: `runtime.SetFinalizer`より柔軟・効率的・安全なファイナライズ機構
- **`testing/synctest`パッケージ**（実験的）: 並行コードのテスト支援、フェイククロックでの時間操作が可能
- **FIPS 140-3対応**: `GOFIPS140`環境変数による暗号モジュール選択
- **ポスト量子暗号**: X25519MLKEM768鍵交換がデフォルトで有効
- **Encrypted Client Hello（ECH）**: TLSサーバーでECHをサポート
- **WebAssembly強化**: `go:wasmexport`ディレクティブ、WASI reactor/libraryとしてのビルドに対応
- **`-json`フラグ**: `go build`, `go install`, `go test`で構造化JSON出力対応

### Go 1.25（2025年8月リリース）

- **コンテナ対応GOMAXPROCS**: Linux cgroup CPU帯域制限を自動検知し、KubernetesなどのCPU制限に応じてGOMAXPROCSを自動調整
- **実験的GC（greenteagc）**: 局所性・並列性・小オブジェクトスキャンの改善。GC負荷の高いワークロードでオーバーヘッド10〜40%削減
- **実験的JSON v2（`encoding/json/v2`）**: `GOEXPERIMENT=jsonv2`で有効化。デコード性能が大幅向上
- **Flight Recorder API**: `runtime/trace.FlightRecorder`により、インメモリのリングバッファで実行トレースを記録
- **DWARF 5デバッグ情報**: バイナリのデバッグ情報サイズ削減、大規模バイナリのリンク時間短縮
- **`testing/synctest`の安定化**: Go 1.24の実験的パッケージが安定版に昇格
- **`go.mod`の`ignore`ディレクティブ**: 特定ディレクトリをgo commandの対象外に指定可能
- **言語仕様**: 「コア型（core types）」概念を仕様から削除し、将来のジェネリクス拡張への布石
- **ASanリーク検出**: `-asan`ビルドでプログラム終了時にメモリリークを自動検出

### Go 1.23（2024年8月リリース）- 振り返り

- **Range-over-Functionイテレータ**: Go 1.22で実験的だった機能が正式に言語仕様に統合。カスタムイテレータを`range`で直接使用可能
- **`iter`標準ライブラリパッケージ**: イテレータの`Seq[T]`型を定義
- **ジェネリクスとイテレータの相乗効果**: Map/Filter等の関数型パターンが容易に実装可能に

## 2. 人気のフレームワーク・ライブラリ

### Webフレームワーク（2025年の利用率）

| フレームワーク | 利用率 | 特徴 |
|---|---|---|
| **Gin** | 48% | 最も人気。高速、豊富なミドルウェア（GitHub 75k+ stars） |
| **Echo** | 16% | ミニマル、高性能ルーティング |
| **Fiber** | 11% | Express風、fasthttp基盤で最高速 |
| **Chi** | - | 軽量、net/http互換 |
| **Beego** | 4% | MVC構造、フルスタック |
| **Encore.go** | - | 分散システム向け、型安全なサービス間通信 |
| **Hertz** | - | 高並行処理特化、マイクロサービス向け |

### ORM・データベース

- **GORM**: 最も普及しているORM
- **sqlx**: database/sqlの実用的な拡張
- **Ent**: Facebookが開発したエンティティフレームワーク、コード生成ベース
- **sqlc**: SQLからGoコードを生成する人気ツール

### 主要ライブラリ

- **ロギング**: `log/slog`（Go 1.21以降の標準）が新規プロジェクトの標準。高性能には`zap`, `zerolog`
- **リンター**: `golangci-lint`が事実上の標準（100以上のリンターを並列実行）
- **RPC**: Buf/Connect（gRPCの代替としてHTTP/1.1対応）
- **イベント駆動**: Watermill
- **テンプレート**: Templ（型安全なHTMLテンプレートエンジン）
- **フルスタック**: HTMX + Go の新潮流

### 標準ライブラリの強さ

- Go 1.22で`net/http`のルーティングが強化され、フレームワークなしでのプロダクション運用がさらに実用的に
- 多くのプロダクションサービスが`net/http`のみで稼働

## 3. クラウドネイティブ / Kubernetesエコシステム

### GoとKubernetes

- Kubernetes、Docker、Terraform、Prometheus、Istio、ArgoCDなど主要CNCFツールの大半がGoで実装
- Goはプラットフォームエンジニアリングの中核言語として確立
- Crossplane、Cilium（eBPF活用）、OpenFaaSなどの新世代ツールもGo製

### Kubernetes最新動向（2025〜2026）

- **Kubernetes v1.34**: Dynamic Resource Allocation（DRA）がGA。GPU等の専用リソース管理がAPIで標準化
- **CNCF Kubernetes AI Conformance**: AIワークロードのポータビリティと相互運用性を検証する適合性テストが正式開始
- **GitOps**: 2026年にはGitOpsが基盤的プラクティスとして定着。Argo CDが主要ツール
- **Kubernetes市場規模**: 2024年の21.1億ドルから2032年に117.8億ドルへ成長予測

### Go 1.25のコンテナ対応

- cgroup CPU制限に基づくGOMAXPROCS自動調整により、Kubernetes環境でのパフォーマンスチューニングが劇的に改善

### 2026年のトレンド

- **プラットフォームエンジニアリング**: Kubernetesの複雑さを抽象化するInternal Developer Platform（IDP）の構築が加速
- **エッジコンピューティング**: 軽量ディストリビューション（K3s等）でGoベースのKubernetesをエッジデバイスに展開
- **FinOps**: AIOps駆動のコスト最適化が主流に
- **セキュリティ**: Kubernetes LTS（長期サポート）リリースの議論が活発化

## 4. AI/MLとGo

### Go製AI/MLインフラツール

- **Ollama**: LLMをローカルで実行するGo製ツール。シンプルなAPIで消費者向けハードウェアに対応（GitHub 130k+ stars）
- **LocalAI**: OpenAI APIのドロップイン代替。GPUなしでLLM/画像生成/音声処理をローカル実行
- **MCP Go SDK**: Model Context Protocolの公式Go SDK

### ML/推論ライブラリ

| ライブラリ | 用途 |
|---|---|
| **GoMLX** | PyTorch/JAX/TensorFlow相当のGo向けMLフレームワーク。ブラウザや組込みデバイスでも動作 |
| **Hugot** | HuggingFace Transformersモデルの推論をGoで実行（ONNX形式） |
| **Go-ONNX v0.19** | ONNX v2.15対応の公式Goバインディング |
| **TensorFlow Go v2.6** | 推論特化の軽量TensorFlowラッパー |
| **ONNX Runtime Go v1.13** | クラウドベースAI推論で広く使用 |
| **Gorgonia** | ディープラーニング向け |
| **Gonum** | 数値計算・一般的ML |

### GoのAI分野でのポジション

- **研究・学習フェーズ**: Pythonが依然として支配的
- **デプロイ・推論フェーズ**: Goが強みを発揮 — 単一バイナリデプロイ、依存関係の簡素化、高並行処理
- **AIツーリング**: gRPC/Kubernetes統合による分散AI推論サービスの構築に最適
- **エージェント/LLMオーケストレーション**: Goの並行処理モデルがAIエージェント構築に適している

## 5. パフォーマンス改善と言語の進化

### パフォーマンス改善の流れ

| バージョン | 改善内容 |
|---|---|
| Go 1.24 | Swiss Tables map、小オブジェクトの効率的メモリ割り当て、新ランタイムmutex → CPU 2-3%削減 |
| Go 1.25 | 実験的GC（greenteagc）→ GC負荷10-40%削減、DWARF 5によるバイナリサイズ削減 |
| Go 1.25 | GOAMD64=v3以上でFMA命令 → 浮動小数点演算高速化・高精度化 |
| Go 1.24-1.25 | プロファイルガイド最適化（PGO）の継続的強化 |

### ジェネリクスの進展

- Go 1.18（2022年）での導入以降、着実に採用が進む
- イテレータとの組み合わせにより、関数型プログラミングパターンが実用的に
- `iterable[T]`制約の提案が進行中（iter.Seq[T], map, chanの統一制約）
- ライブラリ開発者が複雑さを吸収し、一般開発者はシンプルなコードを書ける方向に進化

## 6. コミュニティ・採用トレンド

### 開発者調査（2025年Go Developer Survey）

- **満足度**: 91%が満足（うち約2/3が「非常に満足」）
- **主な用途**: CLIツール（74%）、API/RPCサービス（73%）
- **開発環境**: VS Code（37%）、GoLand/IntelliJ（28%）、Vim/NeoVim（19%）
- **OS**: macOS（60%）、Linux（58%）。デプロイ先はLinux（96%）
- **クラウド**: AWS（46%）、自社サーバー（44%）、GCP（26%）

### AI開発ツールの利用状況

- 53%が毎日AI開発ツールを使用、29%はほぼ未使用
- 満足度は55%だが「非常に満足」はわずか13%
- 主な不満: 非機能的なコード生成（53%）、動くが品質が低い（30%）
- よく使われるツール: ChatGPT（45%）、GitHub Copilot（31%）、Claude Code（25%）、Claude（23%）、Gemini（20%）
- Cursor: Go開発者の6%が主要IDEとして使用

### 採用・人気指標

- **TIOBE Index**: 2025年4月に過去最高の7位
- **GitHub**: 2024年にPython、TypeScriptに次ぐ3番目の成長速度
- **開発者数**: 主言語として220万人、副言語含め500万人超
- **新規採用予定**: 全ソフトウェア開発者の11%が今後12ヶ月以内にGoの採用を計画
- **Cloudflare Radar**: GoがNode.jsを抜き、自動化APIリクエストの最多言語に（12%）

### 開発者の不満トップ3

1. ベストプラクティス/Go慣用句の遵守の難しさ（33%）
2. 他言語にある機能がGoにない（28%）
3. 信頼できるモジュール/パッケージの発見（26%）

## まとめ: 2026年のGoの位置づけ

Go言語は2025〜2026年にかけて、以下の領域で特に強固な地位を占めている：

1. **クラウドネイティブ/インフラの共通言語**として不動の存在
2. **AI推論・デプロイ層**での採用が急速に拡大（Ollama, LocalAI等）
3. **パフォーマンス**はバージョンごとに着実に向上（Swiss Tables, 新GC, PGO）
4. **言語機能**はジェネリクス+イテレータの成熟により表現力が向上しつつ、シンプルさを維持
5. **開発者体験**はAIツール統合が進むが、品質面での課題が残る
6. **エコシステム**はGin/Echo/Fiberが安定した三強、標準ライブラリの強さが際立つ

Goは「シンプルで高速、並行処理に強い」という特性を活かし、AIとクラウドの交差点で独自のポジションを確立している。
