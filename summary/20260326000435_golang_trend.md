# Go言語トレンドレポート (2026年3月)

## 1. 最新バージョンと主要機能

### Go 1.24 (2025年2月)
- **ジェネリック型エイリアス**: 型エイリアスが定義型と同様にパラメータ化可能に
- **`go.mod`のツールディレクティブ**: 実行可能な依存関係を直接追跡可能に（`tools.go`ワークアラウンドが不要に）
- **Swiss Tablesマップ実装**: 新しい組み込みマップ実装により、平均CPU使用量が2-3%削減
- **FIPS 140-3準拠**: `GOFIPS140`で制御される透過的な暗号モジュール
- **ポスト量子暗号**: X25519MLKEM768鍵交換がデフォルトで有効化。`crypto/mlkem`、`crypto/hkdf`、`crypto/pbkdf2`、`crypto/sha3`パッケージ追加
- **Encrypted Client Hello (ECH)** のTLSサポート
- **`go:wasmexport`ディレクティブ**: Go関数をWebAssemblyホストにエクスポート可能に。WASIリアクター/ライブラリビルドのサポート
- **`testing/synctest`** (実験的): フェイククロックを使った並行コードテスト用の隔離されたgoroutine「バブル」
- **`os.Root`型**: ディレクトリ内のサンドボックス化されたファイルシステム操作
- **`testing.B.Loop`**: よりシンプルで高速なベンチマーク
- **JSON `omitzero`タグオプション**: マーシャリング時のゼロ値除外をより正確に制御

### Go 1.25 (2025年8月)
- 言語仕様から「コア型」を削除（簡素化）
- コンテナ認識ランタイム：CPU制限に基づく`GOMAXPROCS`の自動チューニング
- 実験的なGreen Teaガベージコレクタ（`greenteagc`）導入（低ポーズ時間）
- よりスリムで高速なツールチェイン

### Go 1.26 (2026年2月 - 最新リリース、パッチ: 1.26.1, 2026年3月5日)
- **拡張された`new`組み込み関数**: 初期値を指定する式をオペランドとして受け取れるように（例: `p := new(int(42))`）
- **自己参照ジェネリック型**: ジェネリック型が独自の型パラメータリスト内で自身を参照可能に
- **Green Tea GCがデフォルトで有効化**: GC負荷の高いプログラムでGCオーバーヘッドを10-40%削減。最新amd64 CPU（Ice Lake+、Zen 4+）ではベクトル命令ベースのスキャンによりさらに約10%改善。`GOEXPERIMENT=nogreenteagc`でオプトアウト可能
- **cgoオーバーヘッド約30%削減**
- **スライスのスタック割り当て改善**: コンパイラがより多くの状況でスライスバッキングストアをスタックに割り当て
- **実験的`simd/archsimd`パッケージ**: amd64での128/256/512ビットベクトル型（`Int8x16`、`Float64x8`など）によるアーキテクチャ固有のSIMD操作。`GOEXPERIMENT=simd`で有効化
- **実験的`runtime/secret`パッケージ**: 暗号一時データの安全な消去
- **実験的goroutineリークプロファイリング** (`runtime/pprof`)
- **`crypto/hpke`パッケージ**: ポスト量子ハイブリッドKEMサポート付きのHybrid Public Key Encryption (RFC 9180)
- **再構築された`go fix`コマンド**: コードベースを現在のGoイディオムとAPIに自動更新する「モダナイザー」
- **Pprofがフレームグラフビューをデフォルトに**

## 3. 最近追加された主要言語機能

- **ジェネリクスの成熟**: Go 1.24でジェネリック型エイリアス、Go 1.26で自己参照ジェネリック型を追加
- **Range-over-function イテレータ** (Go 1.23): `for x := range f`でイテレータ関数`f`を使用可能に
- **拡張`net/http`ルーティング** (Go 1.22): `http.ServeMux`でのパターンベースルーティング（サードパーティルータの必要性を低減）
- **Swiss Tables** (Go 1.24): より高速な組み込みマップ実装
- **Green Tea GC** (1.25で実験的、1.26でデフォルト): GCパフォーマンスの大幅改善
- **SIMDイントリンシクス** (1.26で実験的): ファーストクラスのベクトル演算
- **コンテナ認識ランタイム** (Go 1.25): cgroup CPU制限からの`GOMAXPROCS`自動チューニング

## 4. Go言語の利用トレンド

### クラウドネイティブ（主力分野）
100万人以上の開発者がクラウド環境でGoを使用。Kubernetes、Docker、Terraform、Prometheus（すべてGoで開発）がエコシステムの礎石として健在。Go多用組織の40%以上がテクノロジーセクターに所属。

### AI/MLインフラストラクチャ
GoはAI研究ではなく、AI**インフラストラクチャ**で強力なニッチを確立:
- **Ollama** (Go製): LLMをローカルで実行するための主要ツール。Llama 3、Mistral、Gemmaなどをサポート
- **LocalAI**: OpenAI互換APIを持つGo製のローカル推論スタック
- **GoMLX**: Hugging Face統合を備えたフル機能MLプラットフォーム
- GoベースのマイクロサービスはPythonベースのサービスと比較して、推論サービングで約30%のレイテンシ改善と約25%のスループット向上を示す（Google Cloud AI Agent Trends 2026）
- Goフレームワークが**Agent2Agent (A2A)**および**Model Context Protocol (MCP)**標準をサポート

### API開発
GoはCloudflare Radar 2024によると、自動APIリクエストで最も人気のある言語としてNode.jsを追い抜き、全APIコールの12%（8.4%から増加）を占有。

### WebAssembly
Go 1.24で`go:wasmexport`とWASIリアクターサポートを追加。**WASIp3**サポートのアクティブな提案が存在し、Wasm内でのイディオマティックなgoroutine使用を可能にする（goroutineがI/Oでブロックしても他をブロックしない）。WASI 1.0は2026年後半〜2027年初頭に予定。

## 5. コミュニティ動向とエコシステムの変化

### 開発者人口
- 過去1年間で410万人のプロがGoを使用。180万人がプライマリ言語として使用（JetBrains）
- 5年前と比較してほぼ倍増

### ランキング
- TIOBE Index **#7**（2025年4月、Go史上最高位）
- GitHub上で3番目に急成長している言語（Python、TypeScriptに次ぐ）

### 2025年Go開発者サーベイ（回答者5,739人）
- **91%がGoに満足**（62%が「非常に満足」）
- トッププロジェクトタイプ: CLIツール (74%)、API/RPCサービス (73%)、ライブラリ (49%)
- AIツール導入は広がるも満足度はまずまず: 55%が満足、しかし53%がAI生成コードが機能しないことを主な問題として挙げる
- 主な不満: ベストプラクティス/イディオムの把握 (33%)、他言語にある機能の欠如 (28%)、信頼できるモジュールの発見 (26%)
- Go開発者の6%が**Cursor**をプライマリIDEとして使用

### 導入意向
全ソフトウェア開発者の11%が今後12ヶ月以内にGoの導入を計画（JetBrains Language Promise Index第4位、TypeScript、Rust、Pythonに次ぐ）

### 主要採用企業
Google、Uber、Netflix、Datadog、Dropbox、HashiCorp、Apple、Salesforce、Amazon、ByteDance、Tencent、Reddit、Pinterest
